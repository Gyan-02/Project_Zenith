## Issue Log

### ISSUE-01 (Hydration mismatch from demo toggle)
- **Severity**: Low
- **Status**: Fixed
- **Expected**: No React hydration mismatch from the demo toggle.
- **Actual**: The first pass showed a stale hydration mismatch because the demo toggle rendered "Live data" on the server and "Demo data" on the first client render.
- **Fix**: `useDemoMode()` now hydrates with a stable initial value and syncs browser-only demo state after mount.

### ISSUE-02 (Share link dropped demo mode)
- **Severity**: Medium
- **Status**: Fixed
- **Expected**: Sharing from `?demo=1` preserves demo mode.
- **Actual**: The copied URL initially dropped `demo=1`, which would open the shared link in live mode.
- **Fix**: share-link generation now preserves only the `demo=1` flag from the current URL before adding location/time/layer state.
